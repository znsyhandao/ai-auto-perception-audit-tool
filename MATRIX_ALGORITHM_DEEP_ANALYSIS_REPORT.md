## 🎯 **核心发现**

### **1. 算法特殊行为**
- 对简单/测试技能返回固定置信度(0.700)
- 有效性标记为"questionable"表示检测不确定
- 可能对复杂技能才有差异化响应

### **2. 数学矛盾解释**
根据公式 `confidence = 0.6 + (density × 0.4)`：
- 如果confidence不变(0.700)，那么density必须不变(0.2500)
- 但测试技能结构不同，density应该不同
- **结论**: 算法对简单技能使用默认/基线值

## 🎯 **核心原则总结**

### **"理解优先于通过"原则**
> **不要为了通过测试而优化，而要为了理解测试而优化。**

**具体含义**:
1. **测试是工具，不是目标** - 测试帮助理解问题，不是要"通过"的障碍
2. **理解检测机制** - 知道测试检测什么，为什么这样检测
3. **优化要有针对性** - 基于对测试的理解进行优化，而不是盲目优化
4. **接受检测局限性** - 有些优化可能无法被特定测试检测到，这本身是重要信息

**应用场景**:
- 当优化没有改善测试结果时，先分析测试机制
- 设计架构时考虑如何能被有效检测
- 建立测试与架构质量的对应关系
- 避免"为测试而测试"的优化循环
