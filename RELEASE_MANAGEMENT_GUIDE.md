# 📦 发布管理指南
## 基于2026-03-29 AISleepGen发布管理教训

## 🎯 目标
建立**专业、一致、无错误的发布管理流程**，确保每次发布都是清晰、可靠、可验证的。

## 📅 创建背景
基于2026-03-29 AISleepGen发布管理问题：
1. **两个重复文件夹**: `v1.0.6/` 和 `latest/` 都是完整副本
2. **CHANGELOG.md版本不一致**: 显示1.0.5，实际1.0.6

## 📋 发布管理原则

### 1. 单一文件夹原则
**原则**: 每个版本只应有一个主文件夹
**实施**:
- 不创建`latest/`符号链接或副本
- 只保留版本特定文件夹：`vX.Y.Z/`
- 发布目录结构：
  ```
  releases/SkillName/
  ├── v1.0.0/          # 历史版本（存档）
  ├── v1.0.1/          # 历史版本
  ├── v1.0.6/          # 当前版本（唯一活动文件夹）
  └── SkillName-v1.0.6.zip  # 发布包
  ```

### 2. 完整版本一致性原则
**原则**: 所有文件版本号必须完全一致
**检查范围**:
- `config.yaml`中的`version`字段
- `package.json`中的`version`字段
- `SKILL.md`中的版本声明
- `CHANGELOG.md`中的最新版本和当前版本声明
- 代码文件中的版本声明（如JS/Python文件）

### 3. 源目录清洁原则
**原则**: 源目录必须干净，不包含发布相关文件
**禁止在源目录中**:
- `latest/`文件夹
- 备份文件 (`*.backup`, `*.bak`, `*.old`)
- 临时检查脚本
- 发布相关配置

## 🛠️ 发布流程

### 阶段1: 发布前准备
```bash
# 1. 更新版本号
# 更新所有文件中的版本号
- config.yaml
- package.json
- SKILL.md
- 代码文件

# 2. 更新CHANGELOG.md
# 添加新版本记录，包含：
- 版本号: [X.Y.Z] - YYYY-MM-DD
- 变更内容
- 修复的问题
- 质量保证状态

# 3. 清理源目录
# 移除临时文件
rm -f *.backup *.bak *.old
rm -rf latest/
```

### 阶段2: 发布检查
```bash
# 运行发布管理检查
python release_management_check.py /path/to/skill

# 运行终极审核
./ultimate_clawhub_audit.ps1 -SkillDir /path/to/skill

# 检查结果要求：
- 总体分数 ≥ 95%
- 无严重问题
- 版本完全一致
```

### 阶段3: 创建发布
```python
# 使用标准化发布脚本
def create_release(skill_dir, release_root, version):
    # 1. 创建版本目录
    version_dir = f"{release_root}/v{version}"
    
    # 2. 复制文件（跳过临时文件）
    copy_skill_files(skill_dir, version_dir)
    
    # 3. 创建ZIP包
    zip_path = f"{release_root}/SkillName-v{version}.zip"
    create_zip(version_dir, zip_path)
    
    # 4. 创建发布说明
    create_release_notes(release_root, version)
    
    # 5. 验证发布
    verify_release(version_dir, zip_path)
```

### 阶段4: 发布后验证
```bash
# 1. 验证文件夹结构
dir releases/SkillName/
# 应该只有: vX.Y.Z/ 和 *.zip

# 2. 验证ZIP包
# 解压并检查内容完整性

# 3. 验证版本一致性
# 检查发布包中所有文件的版本号

# 4. 准备ClawHub提交
# 确保所有材料就绪
```

## 📋 检查清单

### 发布前必须检查：
- [ ] **版本一致性**: 所有文件版本号完全一致
- [ ] **CHANGELOG**: 包含当前版本，最后更新日期正确
- [ ] **文件夹结构**: 无重复文件夹，无备份文件
- [ ] **安全合规**: 通过所有安全检查
- [ ] **文档一致**: 所有声明与代码一致
- [ ] **文件完整**: 所有必需文件存在

### 发布后必须验证：
- [ ] **发布包完整性**: ZIP包可正常解压
- [ ] **文件夹唯一性**: 只有一个版本文件夹
- [ ] **版本正确性**: 发布包中文件版本正确
- [ ] **ClawHub就绪**: 符合所有ClawHub要求

## 💡 经验教训

### 从AISleepGen问题学到的：
1. **简单优于复杂**: 简单的文件夹结构比复杂的符号链接更可靠
2. **自动化检查必须全面**: 不能有任何检查盲点
3. **用户视角很重要**: 用户会注意到我们忽略的细节
4. **文档一致性是关键**: 所有文档必须与代码版本一致
